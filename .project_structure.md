## 📋 **Organization Principles**

### **1. Separation of Concerns**
- **Source code** in `src/`
- **Configuration** in `config/`
- **Data** in `data/`
- **Documentation** in `docs/`
- **Scripts** in `scripts/`

### **2. Environment Separation**
- **Development**: Local files and configurations
- **Production**: Docker containers and Kubernetes
- **Testing**: Isolated test environment

### **3. File Naming Conventions**
- **Scripts**: `snake_case.sh` or `snake_case.py`
- **Modules**: `snake_case.py`
- **Data files**: `descriptive_name.format`
- **Documentation**: `CAPITAL_SNAKE_CASE.md`

### **4. Directory Structure Rules**
- **One responsibility per directory**
- **Logical grouping of related files**
- **Clear separation between environments**
- **Scalable for future expansion**

## 🔧 **Maintenance Guidelines**

### **Adding New Files**
1. **Identify the category** (docs, scripts, data, etc.)
2. **Place in appropriate directory**
3. **Update this documentation**
4. **Test file references**

### **Directory Expansion**
- **New feature**: Create subdirectory in `src/`
- **New data type**: Add to `data/`
- **New documentation**: Add to `docs/`
- **New script**: Add to `scripts/`

### **File Organization Checklist**
- [ ] File in correct directory?
